# DevOps Portfolio

Welcome to my DevOps portfolio. This repository is a collection of projects I have worked on to showcase my skills in various DevOps technologies and practices.

Below you'll find a list of directories, each representing a particular DevOps technology or tool. Each directory contains its own projects along with their respective README files that provide more detailed information about the project and how to use it.

## Table of Contents

- [Ansible](#ansible)
- [Bash Scripts](#bash-scripts)
- [CI-CD Pipelines (Azure Pipelines)](#ci-cd-pipelines)
- [Docker](#docker)
- [Kubernetes](#kubernetes)
- [Python Scripts](#python-scripts)
- [Terraform](#terraform)

### Ansible

In the Ansible directory, you will find a collection of Ansible playbooks and roles that I've created for automating the configuration of systems and deployment of applications.

### Bash Scripts

This directory contains a variety of Bash scripts that I have developed for different tasks, such as system administration, automation, and troubleshooting.

### CI-CD Pipelines

This folder showcases my experience with Continuous Integration and Continuous Deployment. Specifically, it contains YAML files and scripts used to define CI/CD pipelines in Azure Pipelines.

### Docker

The Docker directory consists of Dockerfiles and docker-compose scripts for building and deploying containerized applications.

### Kubernetes

In the Kubernetes directory, you will find YAML and Helm chart files that demonstrate my skills in deploying, scaling, and managing containerized applications using Kubernetes.

### Python Scripts

This directory contains a variety of Python scripts that I have developed for tasks such as automation, web scraping, data manipulation, and API interactions.

### Terraform

The Terraform directory contains .tf files that I have written to provision and manage infrastructure on various cloud providers. It demonstrates my skills in Infrastructure as Code (IaC) using Terraform.

## Getting Started

To get started with any of the projects:

1. Clone this repository to your local machine.
2. Navigate to the directory that corresponds to the technology or tool you're interested in.
3. Follow the instructions in the project's respective README file.
